MetaTrader 5 Error Code Reference

Structured reference of MT5/MQL5 trade return codes, runtime errors, and fixes for algorithmic trading systems.

61 errors
Error Category Cause Fix
ERR_SUCCESS No Error runtime Normal execution. No error occurred. No action required.
ERR_INTERNAL_ERROR Unexpected Internal Error runtime An internal terminal or MQL5 runtime error with no specific cause. Rare. Restart the MetaTrader 5 terminal. Check for terminal updates. Contact your broker if it persists.
ERR_WRONG_INTERNAL_PARAMETER Wrong Internal Parameter runtime Usually caused by a bug in MQL5 source code passing invalid arguments to internal functions. Review function calls. Ensure parameter types match expected types. Check MQL5 documentation for function signatures.
ERR_INVALID_PARAMETER Invalid Parameter runtime A system function was called with an out-of-range or invalid parameter. Validate all inputs before passing to system functions. Check array indices and string lengths.
ERR_NOT_ENOUGH_MEMORY Not Enough Memory runtime The EA or script is consuming too much memory. Large arrays, deep recursion, or memory leaks. Reduce array sizes. Avoid unbounded recursion. Free resources explicitly. Profile memory usage with the MQL5 profiler.
ERR_STRUCT_WITHOBJECTS_ORCLASS Structure or Class with Object runtime Attempting to copy a structure that contains non-copyable objects (e.g. objects with dynamic memory). Use pointers for complex objects in structures. Implement copy constructors where needed.
ERR_INVALID_ARRAY Invalid Array runtime Array index out of bounds or incorrect array size in function calls. Check array sizes before access. Use ArraySize() to validate. Use ArrayResize() to ensure correct dimensions.
ERR_ARRAY_RESIZE_ERROR Array Resize Error runtime ArrayResize() failed because the system cannot allocate the requested memory. Reduce the requested array size. Free other arrays that are no longer needed. Check available system memory.
ERR_STRING_RESIZE_ERROR String Resize Error runtime String concatenation or manipulation ran out of memory. Avoid building very long strings in loops. Use StringConcatenate() carefully and pre-allocate where possible.
ERR_ACCOUNT_DISABLED Account Disabled account The trading account has been disabled by the broker. Contact your broker to enable the account. This is not an EA issue.
ERR_INVALID_ACCOUNT Invalid Account account The account number is invalid or the account does not exist on the server. Verify account credentials and server address. Re-login to the terminal with correct credentials.
ERR_TRADE_TIMEOUT Trade Timeout trade_return The trade request was sent but no response received from the broker server within the timeout period. Check internet connection and server latency. Verify if the order was executed by checking OrdersHistoryTotal(). Implement order confirmation logic to check if the order exists before resending.
ERR_INVALID_PRICE Invalid Price trade_return The price specified in the trade request is invalid (zero, negative, or too far from market). Always use SymbolInfoDouble(symbol, SYMBOL_ASK) for buy orders and SYMBOL_BID for sell orders at the time of order submission. Do not cache prices.
ERR_INVALID_STOPS Invalid Stops trade_return StopLoss or TakeProfit levels are too close to the current price, below the broker's minimum stop level. Check minimum stop level: SymbolInfoInteger(symbol, SYMBOL_TRADE_STOPS_LEVEL). Ensure SL/TP are at least this many points away from the current price.
ERR_INVALID_TRADE_VOLUME Invalid Trade Volume trade_return The lot size is invalid — not between SYMBOL_VOLUME_MIN and SYMBOL_VOLUME_MAX, or not a multiple of SYMBOL_VOLUME_STEP. Use NormalizeDouble(lots, 2) and clamp: lots = MathMax(minLot, MathMin(maxLot, lots)). Check: SymbolInfoDouble(symbol, SYMBOL_VOLUME_MIN/MAX/STEP).
ERR_MARKET_CLOSED Market Closed market Attempting to trade outside of market hours for the symbol. Common for stocks, indices, and some forex pairs on weekends. Check market session hours: SymbolInfoSessionTrade(). Add market hour checks to your EA. Do not send orders on weekends or outside trading sessions.
ERR_TRADE_DISABLED Trade Disabled account Trading is disabled on the account, either by the broker or by the EA setting in the terminal (AutoTrading button disabled). Enable AutoTrading in the MT5 toolbar. Check account permissions: AccountInfoInteger(ACCOUNT_TRADE_ALLOWED). Verify the EA is allowed to trade: TerminalInfoInteger(TERMINAL_TRADE_ALLOWED).
ERR_NOT_ENOUGH_MONEY Not Enough Money account Insufficient free margin to open the position with the specified lot size. Calculate required margin before trading: AccountInfoDouble(ACCOUNT_MARGIN_FREE). Reduce lot size. Add funds to the account. Implement margin check before order submission.
ERR_PRICE_CHANGED Price Changed market The market price moved between when the EA requested the price and when the order was submitted. Common during high volatility. This is normal during fast markets. Use ORDER_TYPE_BUY / ORDER_TYPE_SELL (market orders) instead of limit orders when immediacy is required. Implement retry logic.
ERR_OFF_QUOTES Off Quotes market The symbol has no active market quotes. Happens during weekend, outside trading hours, or when the broker's feed is disconnected. Check if market is active with SymbolInfoInteger(symbol, SYMBOL_TRADE_MODE). Wait for quotes to resume. Add connectivity and session checks to the EA.
ERR_BROKER_BUSY Broker Busy trade_return The broker's trade server is temporarily overloaded and cannot process the request. Implement retry with delay (e.g. Sleep(1000) then retry). Do not spam the server. Check broker status page if this occurs frequently.
ERR_REQUOTE Requote market The price has moved since the request was made and the broker is offering a new price (requote). Set slippage tolerance in the trade request (deviation field). Use ORDER_FILLING_IOC or ORDER_FILLING_FOK based on broker requirements. Consider increasing slippage allowance.
ERR_ORDER_LOCKED Order Locked trade_return The order cannot be modified or closed because it is currently being processed. Wait for the order to finish processing before attempting modification. Implement retry logic with a short delay.
ERR_LONG_POSITIONS_ONLY_ALLOWED Long Positions Only Allowed account The account or symbol only permits long (buy) positions. Short selling is not allowed. Check account type and symbol restrictions. Some retail accounts and certain instruments (e.g. OTC stocks) may not allow shorting.
ERR_TOO_MANY_REQUESTS Too Many Requests trade_return The EA is sending too many trade requests to the server in a short time. Add delays between trade requests (Sleep()). Use a flag to prevent order spamming in OnTick(). Process trade events in OnTradeTransaction() instead of checking every tick.
ERR_TRADE_MODIFY_DENIED Modification Denied trade_return Attempting to modify SL/TP to a level that violates the broker's minimum stop level. Check SYMBOL_TRADE_STOPS_LEVEL and SYMBOL_TRADE_FREEZE_LEVEL before modifying. Ensure new SL/TP meet minimum distance requirements.
ERR_TRADE_CONTEXT_BUSY Trade Context Busy trade_return Another trade operation is in progress. MT5 allows only one trade operation at a time. Use IsTradeAllowed() to check before sending orders. Use a semaphore or flag to serialize trade requests. In MQL5, use Expert Advisor's OnTradeTransaction for confirmation.
ERR_TRADE_EXPIRATION_DENIED Expiration Denied trade_return The broker does not allow pending orders with expiration times. Remove the expiration time from the trade request (set order_expiration to 0). Use ORDER_TIME_GTC instead of ORDER_TIME_SPECIFIED.
ERR_TRADE_TOO_MANY_ORDERS Too Many Orders account Broker-side limit on the number of simultaneously open or pending orders has been reached. Close or cancel some existing orders before opening new ones. Check the limit in account settings. Implement order count checks before placing new orders.
TRADE_RETCODE_REQUOTE Requote (Trade Return Code) trade_return Market moved between request and server processing. Resend with the new price. Increase deviation in the trade request for market orders.
TRADE_RETCODE_REJECT Request Rejected trade_return The trade server rejected the request without a specific reason. Check trade request parameters. Verify account permissions and market conditions. Log the full MqlTradeRequest and MqlTradeResult for debugging.
TRADE_RETCODE_CANCEL Request Cancelled trade_return The trader or automated system cancelled the request before processing. No action needed if intentional. Check EA logic if this is unexpected.
TRADE_RETCODE_PLACED Order Placed trade_return The order has been successfully placed on the server (for pending orders). No action needed. Confirm using OrderSelect() or OnTradeTransaction().
TRADE_RETCODE_DONE Request Done trade_return Trade operation completed as requested. No action needed. Confirm order/position using PositionSelect() or HistoryOrderSelect().
TRADE_RETCODE_DONE_PARTIAL Request Done Partially trade_return Only part of the requested volume was executed. Common with ORDER_FILLING_RETURN fill mode. Check actual filled volume in MqlTradeResult.volume. Decide whether to send a follow-up order for the remaining volume.
TRADE_RETCODE_ERROR Request Processing Error trade_return A generic error during server-side order processing. Log the full result for debugging. Check MqlTradeResult.comment for additional details. Verify all trade request fields are valid.
TRADE_RETCODE_TIMEOUT Request Timed Out trade_return The request was not processed within the allowed time. Check connectivity. Verify if the order was placed by checking order history before resending to avoid duplicate orders.
TRADE_RETCODE_INVALID Invalid Request trade_return The MqlTradeRequest structure contains invalid field values. Validate all fields in MqlTradeRequest. Log the full request for debugging. Check symbol, volume, price, sl, tp, type, and action fields.
TRADE_RETCODE_INVALID_VOLUME Invalid Volume in Request trade_return Volume (lot size) does not meet broker requirements. Normalize volume to valid step: volume = MathFloor(rawLots / lotStep) * lotStep. Check SYMBOL_VOLUME_MIN, MAX, STEP.
TRADE_RETCODE_INVALID_PRICE Invalid Price in Request trade_return The price in the trade request is invalid or stale. Refresh price from SymbolInfoDouble() immediately before order submission.
TRADE_RETCODE_INVALID_STOPS Invalid Stops in Request trade_return SL/TP too close to the current price or violates broker stop levels. Use SYMBOL_TRADE_STOPS_LEVEL to calculate minimum SL/TP distance. Add buffer: stops_level * point * 2.
TRADE_RETCODE_TRADE_DISABLED Trade Disabled for Symbol trade_return Trading is not allowed for the symbol or on the account. Check SYMBOL_TRADE_MODE: SymbolInfoInteger(symbol, SYMBOL_TRADE_MODE). Mode 0 = disabled. Check AutoTrading button in terminal.
TRADE_RETCODE_MARKET_CLOSED Market Closed (Trade Return) trade_return Trade sent outside of market hours. Add session time checks. Use SymbolInfoSessionTrade() to get valid trading sessions before placing orders.
TRADE_RETCODE_NO_MONEY No Money (Trade Return) trade_return Free margin is insufficient for the requested trade size. Calculate required margin before trading. Reduce lot size. Add margin checks: AccountInfoDouble(ACCOUNT_MARGIN_FREE).
TRADE_RETCODE_PRICE_CHANGED Price Changed (Trade Return) trade_return Market moved between order creation and server processing. Retry with fresh price. Use market orders with appropriate deviation setting for fast execution.
TRADE_RETCODE_PRICE_OFF Price Off Quotes trade_return No current market quotes available for the symbol. Check if the symbol has valid quotes: SymbolInfoDouble(symbol, SYMBOL_BID). Wait for market to open or connection to be restored.
TRADE_RETCODE_INVALID_EXPIRATION Invalid Expiration trade_return The expiration time for a pending order is in the past or not allowed by the broker. Use ORDER_TIME_GTC or set expiration to a future time. Check broker's allowed expiration types.
TRADE_RETCODE_ORDER_CHANGED Order State Changed trade_return The order was modified or cancelled externally while being processed. Refresh order state before operating on it. Use OnTradeTransaction() to track order state changes in real time.
TRADE_RETCODE_TOO_MANY_REQUESTS Too Many Requests (Trade Return) trade_return Order rate limit exceeded at the broker level. Add delay between trade operations. Implement a throttle mechanism. Process one trade per OnTick() maximum.
TRADE_RETCODE_NO_CHANGES No Changes trade_return A modification request was sent but the new values are identical to the current values. Compare current order SL/TP with new values before sending modification request to avoid this.
TRADE_RETCODE_SERVER_DISABLES_AT AutoTrading Disabled by Server trade_return The broker's server has disabled automated trading, possibly due to a news event or maintenance. Check broker announcements. Implement TerminalInfoInteger(TERMINAL_TRADE_ALLOWED) and AccountInfoInteger(ACCOUNT_TRADE_ALLOWED) checks.
TRADE_RETCODE_CLIENT_DISABLES_AT AutoTrading Disabled by Client trade_return The AutoTrading button in the MT5 terminal is toggled off. Click the AutoTrading button in the MT5 toolbar to enable it. Check terminal permissions: TerminalInfoInteger(TERMINAL_TRADE_ALLOWED).
TRADE_RETCODE_LOCKED Request Locked trade_return Trade context is busy with another operation. Implement a trade lock mechanism. Wait and retry after a short delay.
TRADE_RETCODE_FROZEN Order Frozen trade_return The order/position is frozen because the market price is within the broker's freeze level range. Wait until the price moves outside the freeze zone. Check SYMBOL_TRADE_FREEZE_LEVEL. Avoid modifying orders during high-volatility events.
TRADE_RETCODE_INVALID_FILL Invalid Fill Mode trade_return The ORDER_FILLING type specified is not supported by the broker or symbol. Check SYMBOL_FILLING_MODE to determine supported fill modes. Use ORDER_FILLING_IOC for most brokers. Test fill modes in a demo account.
TRADE_RETCODE_CONNECTION No Connection to Trade Server connection The terminal lost connection to the broker's trade server. Check internet connection. Verify server address in account settings. Implement connection checks: TerminalInfoInteger(TERMINAL_CONNECTED).
TRADE_RETCODE_ONLY_REAL Real Account Only account Attempting an operation that is only permitted on live accounts, not demo accounts. Test on demo where possible, but note some features (real money withdrawal, certain broker API features) are live-only.
TRADE_RETCODE_LIMIT_ORDERS Pending Order Limit Reached account The broker's maximum number of pending orders has been reached. Cancel unnecessary pending orders before placing new ones. Check the broker's pending order limit.
TRADE_RETCODE_LIMIT_VOLUME Volume Limit Reached account The total open volume for the symbol exceeds the broker's per-symbol or per-account limit. Reduce open positions on the symbol. Check broker limits for maximum volume per symbol. Diversify across symbols if hitting limits.
TRADE_RETCODE_INVALID_ORDER Incorrect or Prohibited Order Type trade_return The order type is not allowed for this symbol or in the current market conditions. Check SYMBOL_TRADE_MODE and SYMBOL_ORDER_MODE. Use only supported order types for the symbol. Some CFDs do not support all order types.
TRADE_RETCODE_POSITION_CLOSED Position Already Closed trade_return Attempting to modify or close a position that no longer exists. Check PositionSelect() before operating on a position. Use OnTradeTransaction() to track position closures in real time.
